A alternativa correta é a c) programação orientada a objetos.

Justificativa:

Estrutura e Interação: O sistema lida com entidades bem definidas que possuem características (atributos) e comportamentos (métodos). A POO permite mapear "Livros", "Autores" e "Usuários" como objetos que interagem entre si, mantendo o estado de cada um (ex: saber se um livro está disponível ou não).

Reuso: Podemos criar uma estrutura base e reutilizá-la. Além disso, a separação em classes permite que o código seja modular; se precisarmos alterar a regra de "Cadastro de Autores", mexemos apenas nessa classe sem afetar o "Fluxo de Empréstimo".

Evolução: A POO facilita a manutenção. Se no futuro a biblioteca passar a oferecer "E-books", podemos usar herança para criar uma classe filha de "Livro" aproveitando o que já existe.

Eficiência: Em um sistema complexo, a POO ajuda a evitar a redundância de dados (como repetir o nome do autor em cada livro) através de associações.
